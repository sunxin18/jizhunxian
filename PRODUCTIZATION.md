# 基准线产品化清单

## 已完成

- 后端存储从 `db.json` 升级为 SQLite：用户、会话、订单分表保存。
- 旧数据自动迁移：服务器已有 `data/db.json` 会在首次启动时迁移到 `data/app.db`。
- 登录收紧：校验中国大陆手机号，服务端会话 30 天过期，登录接口增加基础限流。
- 邮箱验证码：接入 Resend，验证码服务端生成、哈希保存、10 分钟过期、错误次数限制。
- 订单闭环：购买按次服务会创建待支付订单，并在 Pro 页面展示最近订单。
- 安全头：增加 `X-Frame-Options`、`Permissions-Policy`、`X-Content-Type-Options` 等基础响应头。

## 下一步我可以继续做

- 把后端拆成更清晰的 API 层和数据层，补接口测试。
- 增加报告表：保存 AI 复盘、组合体检、回测报告的生成结果和回看入口。
- 增加管理员后台：查看用户、订单、报告、接口错误和数据源状态。
- 接入 Nginx、HTTPS、systemd，把当前 `nohup` 启动升级成稳定生产部署。
- 做基金数据服务端代理和缓存，减少浏览器直连第三方接口的不稳定。

## 需要你配合提供

- 域名：用于 HTTPS、备案和后续微信支付/登录配置。
- 短信服务：阿里云短信、腾讯云短信或其他服务商的签名、模板和密钥。
- 邮件发件域名：Resend 默认发件地址适合测试，正式产品建议配置自己的发件域名。
- 微信支付商户号：商户号、API v3 密钥、证书、回调域名。
- 微信登录能力：公众号/开放平台 AppID、AppSecret、授权回调域名。
- OpenAI 或其他大模型 API Key：用于生成收费报告。
- 法务文案：用户协议、隐私政策、投资风险提示、数据来源说明。
