# 基准线中国大陆上架合规基线

> 目标：基准线要在中国大陆上架和运营，默认按“基金数据观察工具”定位，不做证券基金投资咨询、基金销售、代客理财或自动交易。

## 1. 产品定位红线

- 可以做：基金自选、实时估值展示、历史净值波动、持仓金额记录、提醒、个人复盘、风险提示。
- 谨慎做：基于公开数据的中性描述，例如“波动较大”“单仓集中”“建议复核计划”。
- 不做：明确买入/卖出/加仓/减仓指令、收益承诺、个性化投资建议、基金组合代管、自动交易、荐基收费。
- 页面文案必须保留：数据仅供学习和记录参考，不构成投资建议；估值数据可能延迟或与最终净值不同。

## 2. 上架前必须准备

- Apple Developer Program 账号。
- 中国大陆 App Store 上架所需 ICP/App 备案信息。App Store Connect 里的 ICP 编号和 App 元数据要与工信部备案信息一致。
- 域名和 HTTPS。正式版本不要继续使用 `http://152.136.167.101:8080` 这类裸 IP HTTP 地址。
- 隐私政策 URL、用户协议 URL、儿童/未成年人处理说明、账号注销入口。
- App 内“设置/账号”页一键可查看隐私政策、用户协议、第三方 SDK 清单、个人信息收集清单。

## 3. 个人信息与权限

当前产品涉及的个人信息：

- 账号信息：昵称、手机号或邮箱。
- 登录凭证：服务端 token。
- 用户生成数据：自选基金、持仓金额、提醒规则。
- 技术数据：访问日志、设备/网络信息，若服务器记录需在隐私政策说明。

合规要求：

- 首次启动或首次登录前，显著展示并取得隐私政策同意。
- 只收集实现功能所必需的信息；不默认请求通讯录、定位、相册、麦克风等无关权限。
- 邮箱验证码、手机号登录、持仓同步等目的要写清楚。
- 向第三方提供个人信息或接入 SDK 时，需要在隐私政策和第三方共享清单中说明；必要时单独同意。
- 提供账号注销、导出/删除个人数据、退出登录。

## 4. 金融与基金内容边界

基准线不应描述为：

- “投资顾问”“基金投顾”“荐基”“智能买卖点”“稳赚”“保本”“收益预测”。

更安全的描述：

- “基金实时估值观察工具”
- “自选基金与持仓记录工具”
- “盘中波动提醒与个人复盘工具”

AI 或算法相关文案应避免“预测必然走势”，可使用：

- “基于当前估值和历史净值生成观察摘要”
- “帮助用户复核自己的计划”
- “不替用户作出投资决策”

## 5. 数据源与内容合规

- 明确展示数据来源和延迟/误差提示。
- 不抓取、展示或传播未经授权的付费研报、行情数据或第三方受保护内容。
- 对公开接口依赖要有降级方案，避免误导用户把缓存数据当实时数据。
- 若未来接入推送，推送内容只能做提醒和摘要，不发送明确交易指令。

## 6. 发布检查清单

每次上架或发版前检查：

- [ ] 隐私政策、用户协议、数据源说明可访问。
- [ ] App 内无“买入/卖出/荐基/收益承诺”类文案。
- [ ] 没有未经说明的第三方 SDK。
- [ ] 登录、注销、退出、删除本地缓存路径清楚。
- [ ] iOS 使用 HTTPS 域名，移除开发期 HTTP 放行。
- [ ] App Store Connect 的 App 名称、主体、ICP备案信息一致。
- [ ] 中国大陆合规信息、年龄分级、金融免责声明已填写。

## 7. 需要人工完成的事项

- 办理ICP备案/App备案。
- 准备公司或个人主体材料，确认 App Store Connect 上架主体。
- 找律师或合规顾问复核隐私政策、用户协议、金融内容边界。
- 若未来要做收费、投顾、组合建议或荐基，先确认是否需要持牌机构合作。
